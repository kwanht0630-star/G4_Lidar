# Offline 2D SLAM Blueprint Builder (MATLAB)

This repository contains an offline mapping script that utilizes Simultaneous Localization and Mapping (SLAM) techniques to process moving LiDAR data. It takes the chronological `timetable` of point clouds captured by the Data Acquisition (DAQ) script, calculates the sensor's trajectory through the environment, and stitches the frames together into a clean, unified 2D blueprint.

## 🌟 Features

* **Offline SLAM Processing:** Uses scan-matching algorithms to determine how the sensor moved between frames, automatically aligning and merging consecutive point clouds into a global map.
* **Trajectory Tracking:** Calculates and records the physical path the sensor traveled (`lidarPath`), allowing you to visualize exactly how you walked through the room.
* **Point Cloud Denoising:** Automatically applies `pcdenoise` to every frame before processing, filtering out stray reflections and "ghost" points for a cleaner final map.
* **Blueprint Visualization:** Renders the final output in a clean, high-contrast "Light Theme" figure, designed to look like a professional floor plan or architectural blueprint.

## 📋 Prerequisites

* **Software:** * MATLAB (R2019b or newer)
  * **Navigation Toolbox**
  * **Computer Vision Toolbox** * *Note:* This script relies on `helperLidarMapBuilder`, a standard utility class often provided in MATLAB's Lidar SLAM examples. Ensure this helper file is in your MATLAB path.
* **Data:** A variable named `lidarPointClouds` must exist in your active MATLAB Workspace (generated by the companion Mobile Data Capture script).

## 🚀 Setup and Usage

1. **Capture Data:** First, run the headless Mobile LiDAR Data Capture script and walk the sensor through your environment. 
2. **Verify Workspace:** Ensure the script finished successfully and `lidarPointClouds` is visible in your MATLAB Workspace.
3. **Run the Builder:** Execute this Map Builder script. It will iterate through the saved frames, tracking its progress in the Command Window (e.g., `Starting SLAM mapping for 60 frames...`).

## ⚙️ Key Configuration Parameters

You can fine-tune the SLAM algorithm's accuracy and performance by adjusting the `params` struct at the top of the script:

| Variable | Default | Description |
| :--- | :--- | :--- |
| `DownsampleGridStep` | `0.05` | The voxel filter size (in meters) used to simplify each frame before scan-matching. Higher values process faster but lower mapping accuracy. |
| `RegistrationInlierRatio` | `0.1` | The percentage of points that must match between two frames to be considered a successful alignment. |
| `MergeGridStep` | `0.02` | The voxel size (in meters) used when combining the aligned frames into the final `finalMap`. |

Upon completion, a white-themed MATLAB figure will open containing your environment's reconstructed blueprint.

**Map Key:**
* **Dark Blue Dots (`#2C3E50`):** Detected walls, solid furniture, and physical boundaries.
* **Light Blue Line (`#3498DB`):** The calculated odometry/trajectory (the path the LiDAR sensor traveled during the recording).
* **🟢 Green Circle:** The exact starting position of the scan `(0,0)`.
* **🔴 Red Circle:** The final position of the sensor when the recording ended.
